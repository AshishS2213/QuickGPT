## 🔄 HOW IT WORKS - DATA FLOW

User Input: "todays ipl match"
    ↓ [messageController.js]
    ├─ Receives user message
    │  ↓
[fetchRealtimeDataIfNeeded() called]
    ├─ Detects query type
    │  ↓
[detectQueryType() → returns ['sports']]
    ├─ Found 'ipl' and 'match' keywords
    │  ↓
[fetchSportsData() called]
    ├─ Makes API request to CricketAPI
    │  ↓
CricketAPI Response:
{
  "data": [
    {
      "name": "IPL Match",
      "team1": "Mumbai Indians",
      "team2": "Delhi Capitals",
      "venue": "Dubai Cricket Ground",
      "date": "2026-04-15T19:30:00Z",
      "status": "Scheduled",
      "series": "IPL 2026"
    }
  ]
}
    ├─ Response parsed and mapped
    │  ↓
[formatRealtimeDataForContext() formats it as:]
🏏 SPORTS - CRICKET
   Source: CricketAPI
   • IPL Match
     Mumbai Indians vs Delhi Capitals
     Venue: Dubai Cricket Ground
     Status: Scheduled
     Date: 2026-04-15T19:30:00Z
    │  ↓

    │  ↓

"Today's IPL match is between Mumbai Indians and Delhi Capitals at the Dubai Cricket

## ⚙️ TECHNICAL DETAILS

API Details:
├─ Service: CricketAPI
├─ Base URL: https://api.cricketapi.com/v1/matches
├─ Endpoint: https://api.cricketapi.com/v1/matches?apikey=free&type=live,upcoming
├─ Method: GET
├─ Authentication: None (free tier)
├─ Rate Limit: Sufficient for user queries
├─ Response Format: JSON
├─ Timeout: 5 seconds
└─ Fallback: Built-in with helpful suggestions

Response Structure:
{
  "type": "sports",
  "sport": "cricket",
  "matches": [
    {
      "name": string,
      "series": string,
      "status": string,
      "date": string (ISO 8601),
      "team1": string,
      "team2": string,
      "venue": string,
      "format": string
    }
  ],
  "source": string,
  "timestamp": string
}

Error Handling:
├─ API unavailable → Fallback message with suggestions
├─ Timeout (>5s) → AbortController cancels, returns fallback
├─ Invalid JSON → Caught and logged, fallback provided
├─ Network error → Try-catch returns helpful message
└─ Missing fields → Defaults provided (Team A, Team B, etc.)

Performance:
├─ First query: ~500-800ms (API call)
├─ Cached query: ~1ms (instant)
├─ Cache TTL: Configured in realtimeDataCache.js (1200s for sports)
└─ Parallel execution: Fetches alongside weather, crypto, etc.
